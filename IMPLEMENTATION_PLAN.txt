PROJECT OVERVIEW
================================================================================
The Meesho Image Optimizer is an Expo mobile app that enables sellers to edit 
product images to reduce shipping costs on Meesho and similar e-commerce platforms.

KEY INSIGHT:
Meesho calculates shipping charges based on multiple factors including:
  - Packed product weight (entered by seller)
  - Delivery zone (Local/Regional/National)
  - Image dimensions and file size (used for volumetric weight estimation)
  - Reverse logistics (RTO - Returns to Origin)

By optimizing images (adding padding, adjusting dimensions, changing background),
sellers can potentially reduce the estimated product size/weight, thereby lowering
shipping charges by ₹10-₹40 per order.


MEESHO IMAGE REQUIREMENTS & GUIDELINES (2026)
================================================================================

MANDATORY IMAGE SPECIFICATIONS:
  - Aspect Ratio: 1:1 (Square)
  - Recommended Size: 1000 × 1000 pixels
  - Minimum Size: 500 × 500 pixels
  - Maximum File Size: 10 MB
  - Formats: JPEG or PNG
  - Background: White or light-colored (preferred)

DISALLOWED ELEMENTS:
  - Text overlays
  - Logos or watermarks
  - Borders or collages
  - Blurry or pixelated images
  - Stock photos (must be original)

NUMBER OF IMAGES:
  - Minimum: 3 images per product
  - Maximum: 8 images per product
  - Show: Front, back, close-up, fabric details, size chart (if apparel)

SHIPPING COST OPTIMIZATION TECHNIQUE:
  - Position product smaller and farther in the 1:1 square frame
  - Add extra white space/padding around the product
  - This "tricks" Meesho's algorithm into estimating lower size/weight
  - Avoid close-ups that make items look bulky (increases costs)

   Implementation:
     - Provide rotation controls (0°, 90°, 180°, 270°, or custom angle)

1.4 PADDING & BORDER ADDITION
   Description: Add white space/padding around product to reduce visual size
   Implementation:
     - Preset padding options:

     - Freeform crop tool (user draws crop area)

   def resize_image(image_path, width, height):
       """Resize image to specified dimensions"""
       img = cv2.imread(image_path)
       resized = cv2.resize(img, (width, height), interpolation=cv2.INTER_LANCZOS4)
       return resized

   def rotate_image(image_path, angle):
       """Rotate image by specified angle"""
       img = cv2.imread(image_path)
       h, w = img.shape[:2]
       center = (w // 2, h // 2)
       matrix = cv2.getRotationMatrix2D(center, angle, 1.0)
       rotated = cv2.warpAffine(img, matrix, (w, h))
       return rotated

   def add_padding(image_path, padding_percentage, color=(255, 255, 255)):
       """Add padding around image"""
       img = cv2.imread(image_path)
       h, w = img.shape[:2]
       padding = int(max(h, w) * (padding_percentage / 100))
       padded = cv2.copyMakeBorder(img, padding, padding, padding, padding,
                                   cv2.BORDER_CONSTANT, value=color)
       # Resize to maintain square aspect ratio
       padded = cv2.resize(padded, (1000, 1000), interpolation=cv2.INTER_LANCZOS4)
       return padded

   def remove_background(image_path, background_color=(255, 255, 255)):
       """Remove background and replace with solid color"""
       # Using rembg library
       from rembg import remove
       input_img = Image.open(image_path)
       output_img = remove(input_img)
       # Replace transparent areas with white background
       background = Image.new('RGB', output_img.size, background_color)
       background.paste(output_img, mask=output_img.split()[3])
       return background

   def adjust_brightness(image_path, brightness, contrast, saturation):
       """Adjust brightness, contrast, and saturation"""
       img = cv2.imread(image_path)
       # Brightness adjustment
       img = cv2.convertScaleAbs(img, alpha=1 + (brightness / 100), beta=0)
       # Contrast adjustment (using CLAHE or simple scaling)
       # Saturation adjustment using HSV color space
       return img

PHASE 7: ESTIMATED SHIPPING COST CALCULATION
================================================================================

ALGORITHM:

Meesho uses volumetric weight estimation:
  Volumetric Weight = (Length × Width × Height) / Divisor
  
  Where Divisor typically ranges from 5000-6000 for standard packages

Shipping Cost Factors:
  1. Weight Slab (0-500g, 500g-1kg, 1-1.5kg, 1.5-2kg)
  2. Delivery Zone (Local: ₹35-45, Regional: ₹45-70, National: ₹55-90)
  3. Image Dimensions (used for volumetric estimation)

OPTIMIZATION IMPACT:

Original Image: 1000×1000 px with product taking 80% of frame
  → Estimated volumetric weight: Higher
  → Estimated shipping: ₹80-120 (national)

Optimized Image: 1000×1000 px with product taking 40% of frame (with padding)
  → Estimated volumetric weight: Lower (algorithm sees more white space)
  → Estimated shipping: ₹50-80 (national)
  → Savings: ₹20-40 per order

DISCLAIMER:
  - This is an approximation based on publicly available information
  - Actual shipping costs depend on Meesho's proprietary algorithm
  - Results may vary based on product category and actual package weight
  - Users should test on Meesho seller panel to verify savings

POTENTIAL CHALLENGES & SOLUTIONS
================================================================================

CHALLENGE 1: Background Removal Accuracy
  - Problem: rembg library may not work perfectly on all product types
  - Solution: Implement fallback to manual background selection tool
  - Alternative: Use OpenCV contour detection for simpler products

CHALLENGE 2: Image Processing Performance
  - Problem: Large images (10MB) may cause app lag
  - Solution: Compress images before processing, use worker threads
  - Alternative: Offload heavy processing to backend server

CHALLENGE 3: Cross-Platform Compatibility
  - Problem: Image handling differs between iOS and Android
  - Solution: Use Expo's abstraction layer (expo-image-picker, expo-file-system)
  - Test thoroughly on both platforms

CHALLENGE 4: Shipping Cost Accuracy
  - Problem: Meesho's algorithm is proprietary and not publicly documented
  - Solution: Provide estimated ranges with clear disclaimers
  - Encourage users to test on Meesho seller panel

CHALLENGE 5: Backend Server Stability
  - Problem: Server may crash under heavy load (image processing is CPU-intensive)
  - Solution: Implement request queuing, rate limiting, and error handling
  - Monitor server performance and scale horizontally if needed
